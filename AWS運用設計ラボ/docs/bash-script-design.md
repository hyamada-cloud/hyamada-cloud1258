# Bashスクリプト設計書: system-check.sh

## 1. 文書情報

| 項目 | 内容 |
|---|---|
| 文書名 | Bashスクリプト設計書 |
| 対象スクリプト | `scripts/system-check.sh` |
| 目的 | Linuxサーバ運用を想定した基本状態確認の自動化 |
| 作成区分 | 個人学習・ポートフォリオ |

## 2. 背景

金融系システム運用監視やサーバ運用では、障害発生時にサーバ状態を素早く確認し、状況を整理して関係者へ共有する必要があります。

本スクリプトでは、Linuxサーバの基本確認として、日時、ホスト情報、ディスク使用率、メモリ使用率、任意プロセスの存在確認を行い、結果をログに出力します。

これは実務経験ではなく、Linux運用・Bashスクリプト・運用設計の学習を目的とした個人学習成果物です。

## 3. 目的

- Linuxサーバの基本状態を確認する
- 確認結果をログファイルに記録する
- 正常時と異常時で終了コードを分ける
- 運用手順書・テスト仕様書とセットで管理する
- 面談時に、Bashスクリプトの設計、製造、テストを学習していることを説明できる状態にする

## 4. 対象環境

| 項目 | 内容 |
|---|---|
| OS | WSL2 Ubuntu、Ubuntu Server、Amazon Linux系を想定 |
| シェル | Bash |
| 実行ユーザー | 一般ユーザーを想定 |
| 必要コマンド | `date`, `hostname`, `whoami`, `df`, `free`, `pgrep`, `awk`, `grep`, `cut`, `tee` |

## 5. 非対象

本スクリプトでは、以下の処理は行いません。

- ファイル削除
- サービス停止
- サービス再起動
- ユーザー作成・削除
- AWSリソース作成・変更・削除
- 課金が発生する可能性のある処理

## 6. 処理概要

1. ログ出力先を準備する
2. 実行日時、ホスト名、実行ユーザー、OS情報を出力する
3. ディスク使用率を確認する
4. メモリ使用率を確認する
5. `PROCESS_NAME` が指定されている場合のみ、プロセス存在確認を行う
6. すべて正常であれば `exit 0` で終了する
7. 異常がある場合は `exit 1` で終了する

## 7. 入力値

環境変数で動作を変更できます。

| 変数名 | 初期値 | 説明 |
|---|---:|---|
| `LOG_DIR` | `./logs` | ログ出力先ディレクトリ |
| `LOG_FILE` | `./logs/system-check-YYYYMMDD-HHMMSS.log` | ログファイル名 |
| `DISK_THRESHOLD` | `80` | ディスク使用率の異常判定しきい値 |
| `MEM_THRESHOLD` | `80` | メモリ使用率の異常判定しきい値 |
| `PROCESS_NAME` | 空 | 確認対象プロセス名。空の場合はプロセス確認をスキップ |

## 8. 出力値

### 標準出力

実行結果をコンソールへ表示します。

### ログファイル

`logs/` 配下にログを出力します。

例：

```text
logs/system-check-20260630-090000.log
```

## 9. 終了コード

| 終了コード | 意味 |
|---:|---|
| `0` | 正常終了 |
| `1` | 異常終了 |

異常終了となる主な条件は以下です。

- 必要コマンドが存在しない
- ディスク使用率がしきい値以上
- メモリ使用率がしきい値以上
- 指定したプロセスが存在しない

## 10. 実行例

### 通常実行

```bash
./scripts/system-check.sh
```

### ディスクしきい値を変更して実行

```bash
DISK_THRESHOLD=90 ./scripts/system-check.sh
```

### メモリしきい値を変更して実行

```bash
MEM_THRESHOLD=85 ./scripts/system-check.sh
```

### プロセス確認を行う

```bash
PROCESS_NAME=bash ./scripts/system-check.sh
```

### 異常系テストとして存在しないプロセスを指定する

```bash
PROCESS_NAME=no_such_process ./scripts/system-check.sh
```

## 11. ログ出力設計

| レベル | 用途 |
|---|---|
| `INFO` | 正常な処理、確認結果 |
| `WARN` | 処理継続可能な注意情報 |
| `ERROR` | 異常、終了コード1の原因になる情報 |

## 12. 運用上の注意

- 本スクリプトは確認専用です
- 削除や変更を行う処理は含めていません
- 本番環境で使用する場合は、対象OS、権限、ログ出力先、監視ツール連携を事前に確認する必要があります
- しきい値は環境に合わせて調整が必要です
- 面談では、実務投入済みではなく、個人学習として設計・作成・テストしたものと説明します
