# AWS空調管理・異常監視システム 正式版

#1システムの概要
本システムは、空調機の状態管理、温度・湿度データの登録、異常アラートの確認、定期点検バッチをAWSサーバーレス構成で実現することを目的としている。
Reactで作成した管理画面から空調機情報や温湿度データを登録し、AWS上のAPIを通じてDynamoDBへ保存を行う。
また、EventBridge Schedulerを利用して定期的にLambdaを実行し、DynamoDBに保存されたデータを確認する。
異常条件に該当する場合は、SQSへメッセージを送信し、後続のLambdaでアラート情報を登録する。
この構成により、高額な料金が発生しやすい、EC2やEBSを利用せず、低コストかつサーバーレスで空調管理・異常監視の仕組みを構築できる。

#2AWSシステム構成
S3,CloudFront,API Gateway HTTP API,Lambda,DynamoDB,EventBridge Scheduler,SQS,CloudWatch Logs.

全体の処理の流れは以下の通りです。
1.利用者がブラウザからCloudFront経由でReactアプリへアクセスする
2.ReactアプリはS3に配置された静的ファイルとして配信される
3.認証後、ReactアプリからAPI Gateway HTTP APIへリクエストを送信する
4.API GatewayがLambdaを呼び出す
5.LambdaがDynamoDBへ空調機情報、温湿度データ、アラート情報を保存・取得する
6.EventBridge Schedulerが定期的に点検用Lambdaを起動する
7.点検用LambdaがDynamoDBのデータを確認し、異常があればSQSへ通知メッセージを送信する
8.SQSをトリガーに通知処理Lambdaが実行され、アラート情報をDynamoDBへ保存する
9.各Lambdaの実行ログはCloudWatch Logsに保存される
